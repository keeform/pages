KeeForm Browser Extension Privacy Policy

updated {{ site.data.last_modified.privacy }}

KeeForm handles sensitive data like your usernames, passwords, and URLs. It can also read and change all your data on the websites you visit.

KeeForm uses this information and ability only for one purpose - to fill forms on login pages.

At no time will KeeForm persist, collect, track, transmit, or share your username, password, website URL, or any content of the websites visited - not even in aggregated form.

This is a typical KeeForm workflow:

  1. KeeForm "waits for a message" from KeePass which contains a URL, username and password.
  2. This information is sent to the browser extension using the browser's native messaging protocol.
  3. The KeeForm browser extension opens the provided URL,
  4. scans the web site for username and password fields,
  5. tries to fill those fields with the username and password provided,
  6. and optionally submits the form (disabled by default, and it is recommended to leave it disabled, because the "field finding" algorithm can not be 100% accurate. It might miss form fields, or mistake other form fields as user fields, and so on).

KeeForm discards your sensitive information after a timeout. This time-out is user configurable and by default 30s. The time-out is used in the event of a slow loading website, or if for any reason the form filling process does not complete.

KeeForm does not have access to your KeePass password database.

Important: KeeForm does not actually transmit any form data. Only the website, which was loaded using your URL, determines how and to where your form data is transmitted. Some websites might not secure the transmission properly.
